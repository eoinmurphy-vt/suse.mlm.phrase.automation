在本节中，我们以 openSUSE Leap 为例。

== 准备升级


.过程：升级准备客户端升级
. 在 {productname} 服务器上的命令提示符处，以 root 身份使用 [command]``+spacewalk-common-channels+`` 命令添加相应的通道。
+
----
spacewalk-common-channels \
opensuse_leap15_4 \
opensuse_leap15_4-non-oss \
opensuse_leap15_4-non-oss-updates \
opensuse_leap15_4-updates \
opensuse_leap15_4-uyuni-client
----

. 使用 [command]``+spacewalk-repo-sync+`` 完全同步所有通道。
+
// These are custom channels.
// For more information, see xref:client-configuration:clients-opensuse.adoc[].

. 在 {productname} 服务器 {webui} 中，导航到 menu:软件[管理 > 通道]，并单击 [systemitem]``+Uyuni Client Tools for openSUSE Leap 15.4 (x86_64)+`` 通道名称。

. 在右上角单击 btn:[管理通道]。

. 单击 [guimenu]``+存储库+``+ 选项卡，然后选择 [systemitem]+``+External - Uyuni Client Tools for openSUSE Leap 15.3 (x86_64)+``。

. 单击 btn:[更新存储库]。

. 导航到 menu:存储库[同步] 子选项卡，并单击 btn:[立即同步]。

. 对 [systemitem]``+openSUSE Leap 15.4 (x86_64)+``+ 和 [systemitem]+``+External - openSUSE Leap 15.3 (x86_64)+`` 执行相同的操作。

展开 [systemitem]``+openSUSE Leap 15.4 (x86_64)+`` 以查看填充了软件包的所有子通道。



== 升级

要升级客户端，您需要替换软件储存库，然后更新软件，最后重引导客户端。



.过程：升级升级客户端

. 在 {productname} 服务器 {webui} 中，导航到 menu:系统[]，并单击客户端的名称。
. 单击 menu:软件[软件通道]，并将基础通道选择为在 [systemitem]``+Customs Channels+`` 列表中列出的 openSUSE Leap {opensuse-version} 通道。
. 在 [guimenu]``+子通道+`` 窗格中，选择 {opensuse-version} 子通道。
. 单击 btn:[下一步]，然后单击 [guimenu]``+确认软件通道更改+``，再单击 btn:[确认]。
. 单击 menu:软件[软件包 > 升级]，选择要在客户端上更新的所有软件包，然后应用选择。
  单击 btn:[升级软件包]，检查详细信息，然后单击 btn:[确认] 以完成更新。
. 重引导客户端。

如果您需要更新多个客户端，可以在 {productname} 服务器上创建此命令序列的操作链。
您可以使用操作链同时对多个客户端执行更新。
