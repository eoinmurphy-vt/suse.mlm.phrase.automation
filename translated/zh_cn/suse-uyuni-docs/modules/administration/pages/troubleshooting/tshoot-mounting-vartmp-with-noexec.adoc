[[troubleshooting-mounting-vartmp-with-noexec]]
= 对使用 noexec 挂载 /var/tmp 时出现的问题进行查错
:description: 访问关键信息以在 /var/tmp 的安装点上高效部署服务器，并指定 noexec 选项。
:revdate: 2022-10-18
:page-revdate: {revdate}
:page-author: SUSE 产品与解决方案文档团队
:page-image: https://www.suse.com/assets/img/suse-white-logo-green.svg
:page-product-name: SUSE Multi-Linux Manager
:page-product-version: 5.1

Salt SSH 正在使用 [path]``+/var/tmp+`` 来部署 Salt Bundle，并在客户端上执行带有捆绑 Python 的 Salt 命令。
因此，您不应使用 [option]+``+noexec+`` 选项来挂载 [path]``+/var/tmp+``+。
由于启动过程中使用 Salt SSH 连接客户端，因此无法使用 对那些以 [option]+``+noexec+`` 选项挂载在 [path]``+/var/tmp+``+ 的客户端进行 bootstrap 操作 {webui}。
