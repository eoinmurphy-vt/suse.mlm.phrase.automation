== 目标

作为开发人员，您希望通过构建自定义 StackPack 将您的技术与{stackstate-product-name} 集成。本教程将指导您使用 StackPack CLI 完成整个过程。

== 先决条件

- 访问xref:/setup/cli/cli-sts.adoc[StackPack CLI]
- 启用功能标志：
+
[source,bash]
----
export STS_EXPERIMENTAL_STACKPACK=true
----


== 如何开发自定义集成（StackPack）

=== 搭建新的堆栈包

`+scaffold+` 子命令根据可配置模板创建新的 StackPack 项目结构，简化了 StackPack 开发的初始设置过程。该命令支持本地目录模板和 GitHub 托管模板。

- 使用 CLI 创建一个新的 StackPack 项目：
+
[source,bash]
----
sts stackpack scaffold --name my-stackpack
----

- 使用特定的 GitHub 模板（默认：https://github.com/StackVista/stackpack-templates[stackvista/stackpack-templates]）：

[source,bash]

====

=== 打包您完成的 StackPack 版本

`+package+` 子命令从 stackpack 目录中创建压缩文件。该命令会将所有需要的 stackpack 文件和目录打包成一个正确命名的 zip 压缩包，以便分发和部署。

- 将 StackPack 打包成可发布的压缩包：
+
[source,bash]
----
sts stackpack package -d ./my-stackpack
✅ ✓ Stackpack packaged successfully!

Stackpack: stackpack (v0.0.1)
Zip file: /Users/viliakov/Workspace/src/github/stackstate-cli/my-stackpack/my-stackpack-0.0.1-cli-test.zip
----

- 指定输出文件：
+
[source,bash]
----
sts stackpack package -d ./my-stackpack -f my-custom-archive.zip
----

=== 验证您的堆栈包

在上传和安装 StackPack 之前，可以对其进行验证，以确保配置正确（实现更紧密的反馈回路）。根据您的工作流程，有两种方法：

==== 使用实时后台进行本地验证

`+validate+` 子命令根据运行中的{stackstate-product-name} 实例验证 StackPack。这对于在开发过程中验证 StackPack 非常有用。

- 验证 StackPack 目录（自动打包和上传）：
+
[source,bash]
----
sts stackpack validate -d ./my-stackpack
✅ Stackpack validation successful!
----

- 验证预先打包的`+.sts+` 文件：
+
[source,bash]
----
sts stackpack validate -f ./my-stackpack-0.0.1.sts
✅ Stackpack validation successful!
----

[NOTE]
====
`+sts stackpack validate+` 命令需要在 CLI 上下文中配置一个运行中的{stackstate-product-name} 实例。如果没有可用的实时实例，请使用 CI 验证。
====

==== CI/CD 管道中的验证

对于没有运行{stackstate-product-name} 实例的 CI/CD 管道，可使用 StackState Server Docker 映像中的`+stack-pack-validator+` 工具。这样就可以在不需要实时后台连接的情况下进行验证。

===== GitLab CI 示例

在 GitLab CI 管道中直接使用 StackState Server Docker 镜像：

[source,yaml]
----
.validate-stackpack: &validate-stackpack
  stage: validate
  image: registry.rancher.com/stackstate/stackstate-server:<latest-tag>
  script:
    - |
      echo "Validating stackpack source: stackpacks/$STACKPACK"
      /opt/docker/bin/stack-pack-validator -directory "$CI_PROJECT_DIR/stackpacks/$STACKPACK" || {
        echo "ERROR: Stackpack validation failed for stackpacks/$STACKPACK"
        exit 1
      }
      echo "Stackpack validation passed for stackpacks/$STACKPACK"
  dependencies: []

validate-my-stackpack:
  <<: *validate-stackpack
  variables:
    STACKPACK: my-stackpack
----

===== Docker 命令行

如果在本地或其他 CI/CD 系统中运行验证，请使用`+docker run+` 命令：

- 验证打包的`+.sts+` 文件：
+
[source,bash]
----
docker run --rm --entrypoint="/opt/docker/bin/stack-pack-validator" \
    -v /path/to/my-stackpack-0.0.1.sts:/stackpack.sts \
    registry.rancher.com/stackstate/stackstate-server:<latest-tag> \
    -file /stackpack.sts
----

- 验证堆栈包目录：
+
[source,bash]
----
docker run --rm --entrypoint="/opt/docker/bin/stack-pack-validator" \
    -v /path/to/my-stackpack:/stackpack \
    registry.rancher.com/stackstate/stackstate-server:<latest-tag> \
    -directory /stackpack
----

[TIP]
====
在 CI/CD 管道中使用`+stackstate-server+` 镜像中的`+stack-pack-validator+` 脚本，在发布 StackPacks 之前对其进行验证。这可确保你的 StackPack 满足所有要求，而无需运行{stackstate-product-name} 实例。
====

=== 上传您的堆栈包

`+upload+` 子命令将 zip 压缩包推送到运行中的{stackstate-product-name} 实例。

- 将存档上传至{stackstate-product-name} ：
+
[source,bash]
----
sts stackpack upload -f ./my-stackpack-0.0.1.zip
✅ ✓ Stackpack uploaded successfully!
----
