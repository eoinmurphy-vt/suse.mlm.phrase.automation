== {sle-micro} 5.5 到 {sl-micro} 6.2



本文件提供了经过测试的流程，用于将部署了 {sle-micro} 5.0 服务器的 {productname} 5.5 主机升级到 {sl-micro} 6.2 并迁移到 {productname} {productnumber}。


=== 先决条件

* {productname} 5.0 已安装并在 {sle-micro} 5.5 上运行。
* 系统已注册到 {scc} 且拥有有效的订阅。


=== 发行套件升级和服务器迁移

.过程：升级从 {productname} 5.0 迁移到 {productname} {productnumber}
[role=procedure]
____

. 校验当前产品状态

+

[source,console]
----
SUSEConnect --status-text
----

+

确认以下信息：

* 基础操作系统：`+SUSE Linux Enterprise Micro 5.5+`
* 扩展：`+SUSE Manager Server 5.0 Extension+`

+

. 确保系统已更新

+

[source,console]
----
transactional-update patch
----

+

--

* 如果已应用补丁，请在继续迁移前\\**重引导系统**：

+

[source,console]
----
mgradm stop
reboot
----

* 如果未发现更新，可直接进入迁移步骤。

--

+

. 启动迁移流程：

+

[source,console]
----
transactional-update migration --auto-agree-with-licenses --gpg-auto-import-keys
----

+

按照提示操作，选择可用的迁移目标：**SUSE Linux Micro 6.2** 和 **{productname} 服务器扩展 {productnumber}**。

+

. 清理未使用的容器映像以释放磁盘空间：

+
[source,shell]
----
podman image prune -a
----

+

endif::[]

ifdef::backend-pdf[]
include::../../../../../partials/snippet-check-rpmnew-rpmsave.adoc[]
endif::[]

+

. 停止服务器后重引导，使更改生效。

+

[source,console]
----
mgradm stop
reboot
----

+

. 重引导后的检查：

+

验证 {productname} 扩展和 {productname} 版本：

+

[source,console]
----
SUSEConnect --status-text
mgradm --version
----

+

预期输出：

* 扩展：`+SUSE Multi-Linux Manager Server {productnumber} Extension+`
* 版本：参考 `+5.2.0+` 或更高版本

+

. PostgreSQL 版本

+

[source,console]
----
podman ps
----

+

预期输出：

* `+server:5.2.0+` 或更高版本
* `+server-postgresql:5.2.0+` 或更高版本

+

. 升级服务器容器

+

include::installation-and-upgrade:partial$snippet-ptf-loss-during-upgrade.adoc[]
endif::[]

+

[WARNING]
====
在运行 `+mgradm upgrade podman+` 之前，请检查 `+/etc/systemd/system/uyuni-server.service+` 是否包含格式为 `+-p [::]:PORT:PORT+` 的显式 IPv6 端口绑定。
这些绑定与 {sl-micro} 6.2 上的 netavark 不兼容，并会导致格式错误的 nftables 规则，从而使 Web UI 在升级后无法访问。

如果存在此类行，请在继续之前将其删除：

[source,diff]
----
  -p 80:80 \
- -p [::]:80:80 \
  -p 443:443 \
- -p [::]:443:443 \
----

然后重启主机再继续：

[source,bash]
----
systemctl reboot
----
====

+

[source,console]
----
mgradm start
mgradm upgrade podman
----

+

按照提示操作，拉取并配置新的 5.2.x 容器。

+

. 检查正在运行的容器。

+

[source,console]
----
podman ps
----

+

所有预期的代理容器均应正常启动并运行：

. 清理未使用的容器映像以释放磁盘空间：

+
[source,shell]
----
podman image prune -a
----

+

____


=== 迁移完成

服务器主机系统现在运行的是 {sl-micro} 6.2，并带有更新的 {productname} {productnumber} 服务器软件包。

如果您有一个连接到此服务器的 {productname} 5.0 代理，请参阅 xref:installation-and-upgrade:container-deployment/mlm/migrations/proxy/proxy-mlm-50-52.adoc[代理迁移 5.0 > 5.2] 指南以升级代理主机。

在恢复生产操作之前，请验证您的设置。

== {sles} 15 SP6 升级到 15 SP7

本文件提供了将部署了 {sles} 5.0 服务器的 {productname} 15 SP6 主机升级到 {sles} 15 SP7 并迁移到 {productname} {productnumber} 的程序。


=== 先决条件

* {productname} 5.0 已安装并在 {sles} 15 SP6 上运行。
* 系统已注册到 {scc} 且拥有有效的订阅。


=== 发行套件升级和服务器迁移

.过程：升级在 {productname} 15 SP6 上升级并迁移 {sles}
[role=procedure]
____

. 校验当前产品状态

+

[source,console]
----
SUSEConnect --status-text
----

+

确认以下信息：

* 基础操作系统：`+SUSE Linux Enterprise Server 15 SP6+`
* 扩展：`+SUSE Manager Server 5.0 Extension+`

+

. 确保系统已更新

+

[source,console]
----
zypper patch
----

+

如果已应用补丁，请在继续迁移前停止服务器，然后重引导系统：

+

[source,console]
----
mgradm stop
reboot
----

+

. 执行迁移到 SP7 和 {productname} {productnumber}。

+

[source,console]
----
zypper migration
----

+

选择您的时区。

* `+{sles} 15 SP7+`
* `+{productname} Server Extension {productnumber}+`

+

. 停止服务器后重引导，使更改生效。

+

[source,console]
----
mgradm stop
reboot
----

+

. 重引导后的检查：

+

验证 {productname} 扩展：

+

[source,console]
----
SUSEConnect --status-text
----

+

预期输出：

* 扩展：`+SUSE Multi-Linux Manager Server {productnumber} Extension+`

+

. 验证 {productname} 版本。

+

[source,console]
----
mgradm --version
----

+

预期输出：

* 版本：参考 `+5.2.0+` 或更高版本

+

. 升级服务器容器

+

ifndef::backend-pdf[]
include::installation-and-upgrade:partial$snippet-ptf-loss-during-upgrade.adoc[]
endif::[]

ifdef::backend-pdf[]
include::../../../../../partials/snippet-ptf-loss-during-upgrade.adoc[]
endif::[]

+

[source,console]
----
mgradm start
mgradm upgrade podman
----

+

按照提示操作，拉取并配置新的 5.2.x 容器。


. 更新容器

+

[source,console]
----
podman ps
----

+

预期输出：

* `+server:5.2.0+` 或更高版本
* `+server-postgresql:5.2.0+` 或更高版本

____


=== 迁移完成

服务器主机系统现在运行 {sles} 15 SP7，并带有更新的 {productname} {productnumber} 服务器软件包。

如果您有一个连接到此服务器的 {productname} 5.0 代理，请参阅 xref:installation-and-upgrade:container-deployment/mlm/migrations/proxy/proxy-mlm-50-52.adoc[代理迁移 5.0 > 5.2] 指南以升级代理主机。

在恢复生产操作之前，请验证您的设置。
