== {sle-micro} 5.5에서 {sl-micro} 6.2로



이 문서는 {sle-micro} 5.0 서버에서 배포된 {productname} 5.5 호스트를 {sl-micro} 6.2로 업그레이드하고 {productname} {productnumber}로 마이그레이션하는 테스트된 절차를 제공합니다.


=== 전제 조건

* {productname} 5.0이 {sle-micro} 5.5에 설치되어 실행 중입니다.
* 시스템은 {scc}에 등록되어 있으며 활성 구독을 보유하고 있습니다.


=== 업그레이드 및 마이그레이션

.절차 {productname} 5.0에서 {productname} {productnumber}로의 마이그레이션
[role=procedure]
____

. 현재 제품 상태를 확인합니다.

+

[source,console]
----
SUSEConnect --status-text
----

+

확인:

* Base OS: `+SUSE Linux Enterprise Micro 5.5+`
* 확장 프로그램: `+SUSE Manager Server 5.0 Extension+`

+

. 시스템이 업데이트되었는지 확인합니다.

+

[source,console]
----
transactional-update patch
----

+

--

* 패치가 적용된 경우 마이그레이션을 진행하기 전에 서버를 중지한 다음 시스템을 재부팅합니다.

+

[source,console]
----
mgradm stop
reboot
----

* 업데이트가 없는 경우, 마이그레이션 단계로 바로 진행할 수 있습니다.

--

+

. 업그레이드를 시작합니다.

+

[source,console]
----
transactional-update migration --auto-agree-with-licenses --gpg-auto-import-keys
----

+

안내에 따라 **SUSE Linux Micro 6.2** 및 **{productname} 서버 확장 프로그램 {productnumber}**로 가능한 마이그레이션을 선택합니다.

+

. 사용하지 않는 컨테이너 이미지를 정리하여 디스크 공간을 확보합니다.

+
[source,shell]
----
podman image prune -a
----

+

+

. 서버를 중지한 다음 재부팅하여 변경 사항을 적용합니다.

+

[source,console]
----
mgradm stop

----

+

. 재부팅 후 점검을 수행합니다.

+

{productname} 확장 프로그램 및 {productname} 버전 확인:

+

[source,console]
----
SUSEConnect --status-text
mgradm --version
----

+

예상 출력:

* 확장 프로그램: `+SUSE Multi-Linux Manager Server {productnumber} Extension+`
* 버전: `+5.2.0+` 이상 참조

+

. PostgreSQL 버전

+

[source,console]
----
podman ps
----

+

예상 출력:

* `+server:5.2.0+` 이상
* `+server-postgresql:5.2.0+` 이상

+

+

+

[WARNING]
====
`+mgradm upgrade podman+`을(를) 실행하기 전에 `+/etc/systemd/system/uyuni-server.service+`에 `+-p [::]:PORT:PORT+` 형식의 명시적 IPv6 포트 바인딩이 포함되어 있는지 확인하십시오.
이러한 바인딩은 {sl-micro} 6.2의 netavark와 호환되지 않으며 업그레이드 후 웹 UI에 연결할 수 없게 만드는 잘못된 nftables 규칙을 유발합니다.

이러한 줄이 있는 경우, 진행하기 전에 제거하십시오:

[source,diff]
----
  -p 80:80 \
- -p [::]:80:80 \
  -p 443:443 \
- -p [::]:443:443 \
----

그런 다음 계속하기 전에 호스트를 재부팅하십시오:

[source,bash]
----
systemctl reboot
----
====

+

[source,console]
----
mgradm start
mgradm upgrade podman
----

+

안내에 따라 새로운 5.2.x 컨테이너를 가져오고 구성하십시오.

+

. 실행 중인 컨테이너를 확인합니다.

+

[source,console]
----
podman ps
----

+

예상되는 모든 프록시 컨테이너가 작동 중이어야 합니다.

. 사용하지 않는 컨테이너 이미지를 정리하여 디스크 공간을 확보합니다.

+
[source,shell]
----
podman image prune -a
----

+

=== 마이그레이션

서버 호스트 시스템이 이제 업데이트된 {productname} {productnumber} 서버 패키지와 함께 {sl-micro} 6.2를 실행 중입니다.

이 서버에 연결된 {productname} 5.0 프록시가 있는 경우, 프록시 호스트를 업그레이드하기 위해 xref:installation-and-upgrade:container-deployment/mlm/migrations/proxy/proxy-mlm-50-52.adoc[Proxy Migration 5.0 > 5.2] 가이드로 진행하십시오.

생산 작업을 재개하기 전에 설정을 검증하십시오.
