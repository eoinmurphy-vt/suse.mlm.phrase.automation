[[troubleshooting-salt-ssh-known-hosts]]
= Salt SSH 알려진 호스트 문제 해결
:revdate: 2026-04-15
:page-revdate: {revdate}

머신이나 VM이 새로운 SSH 키로 재설치되었지만 동일한 IP 주소를 사용하여 온보딩하는 경우, Salt가 \"중간자 공격(man-in-the-middle attack)\"에 대해 경고할 가능성이 있습니다.
이는 Salt의 `+.ssh/known_hosts+` 파일에 여전히 이전 SSH 지문이 포함되어 있기 때문에 발생합니다.

이 목록에서 호스트를 제거하려면 다음 명령을 실행하십시오.

[source,shell]
----
mgrctl ssh knownhost remove <hostname>
----

제거할 호스트가 기본 SSH 포트 `+22+`을(를) 사용하지 않는 경우, 포트를 수동으로 지정할 수 있습니다.

[source,shell]
----
mgrctl ssh knownhost remove <hostname> <port>
----
