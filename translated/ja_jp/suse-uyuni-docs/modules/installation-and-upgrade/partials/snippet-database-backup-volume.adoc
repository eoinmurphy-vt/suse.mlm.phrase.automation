= データベースバックアップボリューム
:revdate: 2025-07-29
:page-revdate: {revdate}

[command]``+mgradm migration+``または[command]``+mgradm upgrade+``を使用したサーバ移行またはアップグレードにより、データベースのバックアップを含むボリュームを作成できます。

{postgresql}データベースのバージョンが上がると、アップグレードを実行する前に古いデータベースを別の場所に保存する必要があります。
この目的のために[command]``+mgradm+``は、ボリューム[パス]``+var-pgsql-backup+``を動的に作成します。
移行またはアップグレードが完了し、ユーザーが新しいシステムが期待通りに動作していることを確認したら、このボリュームは安全に削除することができます。
