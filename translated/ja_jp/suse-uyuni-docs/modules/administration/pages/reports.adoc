= レポートの生成
:description: SUSE Multi-Linux ManagerのHub Reportingツールを使用してカスタマイズされたレポートを生成する方法を学び、その後、サポートするレポーティングツールに接続します。
:revdate: 2024-05-08
:page-revdate: {revdate}
:page-author: SUSE製品およびソリューションドキュメンテーションチーム
:page-image: https://www.suse.com/assets/img/suse-white-logo-green.svg
:page-product-name: SUSE Multi-Linux Manager
:page-product-version: 5.1

{productname}は、ユーザーがさまざまなレポートを生成できるようにします。
これらのレポートは、登録されたシステム、ユーザー、および組織の在庫を取るのに役立ちます。
レポートを使用することは、特に管理下に多くのシステムがある場合、{productname} {webui}から手動で情報を収集するよりも簡単なことがよくあります。

コマンドラインツール[command]``+spacewalk-report+``を使用して事前に構成されたレポートを生成できますが、xref:specialized-guides:large-deployments/hub-reporting.adoc[Hub Reporting]の導入により、完全にカスタマイズされたレポートを生成することも可能です。
これは、SQL言語をサポートする任意のレポーティングツールをレポーティングデータベースに接続し、データを直接抽出することで実現できます。
データの可用性と構造に関する詳細については、レポーティングデータベーススキーマのドキュメントを参照してください。

== ``+spacewalk-report+``を使用する


[IMPORTANT]
====
サーバコンテナ内でステップを実行する前に、[literal]``+mgrctl term+``を使用してください。
====

レポートを生成するには、[package]``+spacewalk-reports+``パッケージがインストールされている必要があります。
[command]``+spacewalk-report+``コマンドを使用すると、{productname}全体のコンテンツ、システム、およびユーザーリソースに関するレポートを整理して表示できます。

[IMPORTANT]
====
xref:specialized-guides:large-deployments/hub-reporting.adoc[Hub Reporting]の導入により、[command]``+spacewalk-report+``はデフォルトでレポーティングデータベースからデータを収集します。詳細については、<<spacewalk-report-reporting-db>>を参照してください。
====

次に関するレポートを生成できます。

システムインベントリ::
{productname}に登録されているすべてのシステムを一覧にします。

パッチ:: 
登録されているシステムに関連するすべてのパッチを一覧にします。
重大度、および特定のパッチに適用されるシステムでパッチをソートできます。

ユーザー:: 
すべての登録済みユーザおよび特定のユーザに関連するすべてのシステムを一覧にします。

CSV形式でレポートを取得するには、サーバのコマンドプロンプトで次のコマンドを実行します。

[source]
----
spacewalk-report <report_name>
----

== ``+spacewalk-report+``およびレポーティングデータベース

[command]``+spacewalk-report+``はデフォルトで新しいレポーティングデータベースを使用してデータを抽出します。
これは、新しく生成されたレポートがデータの構造と形式にいくつかの違いを持つことを意味します。
すべてのレポートに共通する違いは次のとおりです：

- レポートデータはリアルタイムでは変更されず、スケジュールされたタスクの実行によってのみ更新されます。
- データの重複が削除され、以前は「multival」と見なされていた列には、現在``+;+``+で区切られた複数の値が含まれています。これにより、コマンドラインオプション+``+--multival-on-rows+``+および+``+--multival-separator+``は新しいレポートには適用されなくなります。なぜなら、その動作が現在のデフォルトだからです。
- すべてのレポートで、新しい列``+mgm_id+``+および``+synced_date+``が導入され、ハブシナリオの管理システムと、アプリケーションデータベースから情報が最後に更新された時刻を識別します。
- すべてのブール値は、``+True+``+/+``+False+``+で表されるようになり、+``+1+``+/+``+0+``の値ではなくなりました。
- 列``+org_id+``+は、数値識別子ではなく組織名を含む+``+organization+``に置き換えられました。
- 「サーバー」という用語は「システム」に置き換えられました。したがって、例えば、列``+server_id+``+は現在+``+system_id+``と呼ばれています。

レポート固有の変更については、<<report-list>>を参照してください。

[IMPORTANT]
====
この変更された動作によって問題が発生した場合、新しいオプション``+--legacy-report+``を使用して、アプリケーションデータベースに対して実行される古いレポートにフォールバックできます。
====

ハブレポートに関する詳細については、xref:specialized-guides:large-deployments/hub-reporting.adoc[]を参照してください。

個別のレポートに関する詳細情報については、[command]``+spacewalk-report+``+を実行し、オプション[option]+``+--info+``+または[option]+``+--list-fields-info+``とレポート名を指定してください。
これにより、レポート内の説明と可能なフィールドのリストが表示されます。

プログラムの呼び出しとオプションの詳細については、[literal]``+spacewalk-report(8)+``+マニュアルページおよび[コマンド]+[option]+``+--help+``+パラメータ、さらに[コマンド]+``+spacewalk-report+``コマンドも参照してください。
