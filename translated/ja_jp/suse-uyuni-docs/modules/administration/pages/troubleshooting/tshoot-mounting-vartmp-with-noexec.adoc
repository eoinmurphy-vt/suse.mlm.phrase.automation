[[troubleshooting-mounting-vartmp-with-noexec]]
= noexecで/var/tmpをマウントする場合のトラブルシューティング
:description: サーバーを効率的にデプロイするために、/var/tmpのマウントポイントにnoexecオプションを指定して、アクセスキー情報を取得してください。
:revdate: 2022-10-18
:page-revdate: {revdate}
:page-author: SUSE製品およびソリューションドキュメンテーションチーム
:page-image: https://www.suse.com/assets/img/suse-white-logo-green.svg
:page-product-name: SUSE Multi-Linux Manager
:page-product-version: 5.1

Salt SSHは、クライアントにSalt Bundleをデプロイし、バンドルされたPythonでSaltコマンドを実行するために、[path]``+/var/tmp+``を使用しています。
したがって、[path]``+/var/tmp+``+を[option]+``+noexec+``オプションでマウントしてはいけません。
クライアントが[path]``+/var/tmp+``+を[option]+``+noexec+``オプションでマウントしている場合、起動プロセスがSalt SSHを使用してクライアントに到達するため、{webui}でクライアントを起動することはできません。
