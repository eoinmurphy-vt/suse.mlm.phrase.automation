= プロキシのアップグレード（5.1から5.2へ）
:revdate: 2026-06-30
:page-revdate: {revdate}

== 概要


このドキュメントは、**{productname}**で管理される環境において、**ホストオペレーティングシステム**と**プロキシ拡張機能**の両方を移行するためのテスト済みかつ検証済みのプロシージャを提供します。特に、**{productname} Proxy 5.1**がデプロイされたシステムを対象としています。

対象となるアップグレードシナリオは次のとおりです。

* **SL Micro 6.1**から**SL Micro 6.2**への移行
* **SLES 15 SP7**を新しい**{productname}プロキシ拡張機能 {productnumber}**で更新します。
* **{productname}プロキシ拡張機能**を**バージョン 5.1**から**バージョン {productnumber}**にアップグレードします。

[IMPORTANT]
====
プロキシを移行する前に、まず{productname} 5.1サーバを{productname} {productnumber}に移行する必要があります。
====



== {sl-micro} 6.1から{sl-micro} 6.2

このセクションでは、{productname} 5.1プロキシでデプロイされた{sl-micro} 6.1ホストを、{productname} {productnumber}プロキシを備えた{sl-micro} 6.2にアップグレードするためのテスト済みの手順を提供します。


=== 前提条件

* {productname} 5.1プロキシが{sl-micro} 6.1でインストールされ、実行中です。
* プロキシシステムが{productname}サーバに登録されています。


=== ディストリビューションのアップグレードとプロキシの移行

.プロシージャ:{productname} 5.1プロキシを{productname} {productnumber}プロキシに移行します。
[role=procedure]
____

. システムと{productname}ツールのバージョンを確認します。

+

[source,console]
----
cat /etc/os-release
mgrpxy --version
----

+

確認:

* オペレーティングシステム: `+SL Micro 6.1+`
* ツールバージョン: `+mgrpxy version 5.1.x+`以上

+

. 実行中のコンテナを確認する:

+

[source,console]
----
podman ps
----

+

以下のコンテナが実行されていることを確認します。

- `+proxy-squid+`
- `+proxy-ssh+`
- `+proxy-httpd+`
- `+proxy-tftpd+`
- `+proxy-salt-broker+`

+

. {productname}サーバに新しいプロキシ製品を同期します。
  詳細については、xref:client-configuration:products.adoc[]を参照してください。

+

Web UIを使用して同期します。

* `+SL Micro 6.2+`
* `+{productname} Proxy Extension {productnumber}+`

+

. 移行の実行
  詳細については、xref:client-configuration:client-upgrades-product-migration.adoc[]を参照してください。

+

プロキシシステムに移動し、menu:システム[概要] > menu:ソフトウェア[製品移行]を選択します。

+

移行元

* `+SL Micro 6.1+` + `+SUSE Multi-Linux Manager Proxy 5.1 Extension+`

+

移行先

* `+SL Micro 6.2+` + `+{productname} Proxy Extension {productnumber}+`

+

[IMPORTANT]
====
プロンプトが表示されたときに、必要であることが確認されていない限り、オプションのチャンネルを**選択しないでください**。
====

+

[NOTE]
====
アップグレードする前に、サーバのバックアップを作成することをお勧めします。
====

+

. 移行アクションを監視する

+

menu:システム[詳細> イベント]タブのでプロセスを追跡できます{webui}。

+

. アップグレードが完了したら、システムを再起動する:

+

[source,console]
----
mgrpxy stop
reboot
----

+

. 再起動後のチェック:

+

アップグレードされたOSと{productname}拡張機能を確認してください。

+

[source,console]
----
cat /etc/os-release
SUSEConnect --status-text
----

+

. {productname}ツールのバージョンを確認してください。

+

[source,console]
----
mgrpxy --version
----

+

予想される出力:

- `+mgrpxy version 5.2.0+`以上

+

. RPMパッケージとして新しいプロキシコンテナイメージをインストールします。

+

[source,shell]
----
transactional-update pkg install suse-multi-linux-manager-5.2-<arch>-proxy*
----

+

. プロキシを再起動します。

. プロキシコンテナをアップグレードします。

+

[source,console]
----
mgrpxy upgrade podman
mgrpxy stop
mgrpxy start
----

+

エアギャップ(された)環境でのインストールでは、まずコンテナのRPMパッケージをアップグレードし、その後`+mgrpxy upgrade podman+`コマンドを実行します。

+

または、Kubernetesクラスタ上で実行されるプロキシコンテナは、次のコマンドを使用して更新できます。

+

[source,shell]
----
mgrpxy upgrade kubernetes
----

+

. 未使用のコンテナイメージをクリーンアップし、ディスク容量を解放します。

+
[source,shell]
----
podman image prune -a
----

+

+

. プロキシコンテナが動作していることを確認する

+

[source,console]
----
podman ps
----

+

すべての予期されるプロキシコンテナが稼働中である必要があります。

- `+proxy-salt-broker+`
- `+proxy-httpd+`
- `+proxy-squid+`
- `+proxy-tftpd+`
- `+proxy-ssh+`

=== 移行の完了

プロキシホストシステムは、{sl-micro} 6.2を実行しており、更新された{productname} {productnumber}プロキシパッケージと同期された製品チャンネルを備えています。


== {sles} 15 SP7

このセクションでは、{productname} 5.1プロキシでデプロイされた{sles} 15 SP7ホストを{productname} {productnumber}プロキシに更新するための手順を提供します。


=== 前提条件

* {productname}プロキシ5.1は、{sles} 15 SP7にインストールされ、実行されています。
* プロキシシステムが{productname}サーバに登録されています。


=== プロキシコンポーネントの更新

.プロシージャ:{sles} 15 SP7で{productname}プロキシコンポーネントを更新します。
[role=procedure]
____

. オペレーティングシステムと{productname}ツールのバージョンを確認してください。

+

[source,console]
----
cat /etc/os-release
mgrpxy --version
----

+

確認:

* オペレーティングシステム: `+SUSE Linux Enterprise Server 15 SP7+`
* ツールバージョン: `+mgrpxy version 5.1.x+`以上


+

. 実行中のプロキシコンテナを一覧表示する

+

[source,console]
----
podman ps
----

+

以下のコンテナが実行されていることを確認します。

- `+proxy-salt-broker+`
- `+proxy-httpd+`
- `+proxy-squid+`
- `+proxy-tftpd+`
- `+proxy-ssh+`

+

. Web UIを使用して、{productname}サーバ内の新しい製品を同期します。

+

詳細については、xref:client-configuration:products.adoc[]を参照してください。

+

同期するディレクトリ:

* `+{productname} Proxy Extension {productnumber}+`

+

. 移行の実行

+

詳細については、xref:client-configuration:client-upgrades-product-migration.adoc[]を参照してください。

+

プロキシシステムに移動し、menu:システム[概要] > menu:ソフトウェア[製品移行]を選択します。

+

移行元

* `+{sles} 15 SP7+` + `+SUSE Multi-Linux Manager Proxy 5.1 Extension+`

+

移行先

* `+{sles} 15 SP7+` + `+{productname} Proxy Extension {productnumber}+`

+

[IMPORTANT]
====
プロンプトが表示されたときに、必要であることが確認されていない限り、オプションのチャンネルを**選択しないでください**。
====

+

[NOTE]
====
アップグレードする前に、サーバのバックアップを作成することをお勧めします。
====

+

. 移行アクションを監視する

+

{webui}のmenu:システム[詳細> イベント]で処理を追跡できます。

+

. アップグレードが完了したら、システムを再起動する:

+

[source,console]
----

----

+

. 再起動後のチェック:

+

OSおよび{productname}拡張機能のバージョンを確認します：

+

[source,console]
----
cat /etc/os-release

----

+

. {productname}ツールのバージョンを確認してください。

+

[source,console]

----

+

予想される出力:

- `+mgrpxy version 5.2.0+`以上

+

. RPMパッケージとして新しいプロキシコンテナイメージをインストールします。

+

[source,shell]
----

----

+

. プロキシを再起動します。

. プロキシコンテナをアップグレードします。

----

エアギャップ(された)環境でのインストールでは、まずコンテナのRPMパッケージをアップグレードし、その後`+mgrpxy upgrade podman+`コマンドを実行します。
