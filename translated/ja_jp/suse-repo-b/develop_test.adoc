== ゴール

開発者として、あなたはカスタムStackPackを構築することによって、自分の技術を{stackstate-product-name} 。このチュートリアルでは、StackPack CLIを使用した手順を説明します。

== 前提条件

- xref:/setup/cli/cli-sts.adoc[StackPack CLIへの]アクセス
- 機能フラグが有効：
+
[source,bash]
----
export STS_EXPERIMENTAL_STACKPACK=true
----


== カスタム・インテグレーション（StackPack）の開発方法

=== 新しいStackPackを足場にする

`+scaffold+` サブコマンドは、設定可能なテンプレートから新しい StackPack プロジェクト構造を作成し、StackPack 開発の初期設定プロセスを合理化します。このコマンドは、ローカル・ディレクトリのテンプレートとGitHubでホストされているテンプレートの両方をサポートしている。

- CLIを使用して新しいStackPackプロジェクトを作成します：
+
[source,bash]
----
sts stackpack scaffold --name my-stackpack
----

- 特定のGitHubテンプレートを使用する場合（デフォルト https://github.com/StackVista/stackpack-templates[：]）：
+
[source,bash]
----

=== レビューとカスタマイズ

- 生成されたファイルをターゲット・ディレクトリで確認する。
- `+stackpack.yaml+` などのファイルを編集して、統合ロジックを定義します。
  * データを統合する。コンポーネントとインジェスト・メトリクスのマッピングについては、「xref:setup/custom-integrations/otelmappings/README.adoc[Otel Telemetryマッピングの追加]」を参照してください。

  * xref:setup/custom-integrations/metric-bindings/index.adoc[メートルバインディングの追加]

[NOTE]
====
default:https://github.com/StackVista/stackpack-templates[stackvista/stackpack-templates]テンプレートは、モニターやメトリックバインディングなど、ほとんどの拡張ポイントの例が含まれているため、出発点として最適です。

- `+test+` サブコマンドは、あなたのStackPackをパッケージ化し、アップロードし、テストバージョンの接尾辞を付けてインストールします。

=== パッケージ StackPackの完成バージョン

`+package+` サブコマンドはstackpackディレクトリからzipファイルを作成する。このコマンドは、必要なすべてのstackpackファイルとディレクトリを、配布と配備のために適切な名前のzipアーカイブにパッケージ化します。

- StackPackを配布可能なzipにパッケージします：
+
[source,bash]
----
sts stackpack package -d ./my-stackpack
✅ ✓ Stackpack packaged successfully!

Stackpack: stackpack (v0.0.1)
Zip file: /Users/viliakov/Workspace/src/github/stackstate-cli/my-stackpack/my-stackpack-0.0.1-cli-test.zip
----

- 出力ファイルを指定する：
+
[source,bash]
----
sts stackpack package -d ./my-stackpack -f my-custom-archive.zip
----

=== StackPackを検証する

StackPackをアップロードしてインストールする前に、それが適切に設定されていることを確認するために検証することができます（よりタイトなフィードバックループのために）。ワークフローによって2つのアプローチがある：

==== ライブ・バックエンドによるローカル検証

`+validate+` サブコマンドは、実行中の{stackstate-product-name} インスタンスに対して StackPack を検証する。これは、開発中にStackPackを検証するのに便利です。

- StackPackディレクトリを検証する（自動的にパッケージ化され、アップロードされる）：
+
[source,bash]
----
sts stackpack validate -d ./my-stackpack
✅ Stackpack validation successful!
----

- 事前にパッケージ化された`+.sts+` ファイルを検証する：
+
[source,bash]
----
sts stackpack validate -f ./my-stackpack-0.0.1.sts
✅ Stackpack validation successful!
----

[NOTE]
====
`+sts stackpack validate+` コマンドを実行するには、CLI コンテキストで構成された実行中の{stackstate-product-name} インスタンスが必要です。ライブ・インスタンスが利用できない場合は、代わりにCIバリデーションを使用する。
====

==== CI/CDパイプラインにおけるバリデーション

実行中の{stackstate-product-name} インスタンスがないCI/CDパイプラインの場合は、StackState Server Dockerイメージの`+stack-pack-validator+` ツールを使用します。これは、ライブのバックエンド接続を必要とせずに検証を実行します。

===== GitLab CI の例

GitLab CIパイプラインでStackState ServerのDockerイメージを直接使用する：

[source,yaml]
----
.validate-stackpack: &validate-stackpack
  stage: validate
  image: registry.rancher.com/stackstate/stackstate-server:<latest-tag>
  script:
    - |
      echo "Validating stackpack source: stackpacks/$STACKPACK"
      /opt/docker/bin/stack-pack-validator -directory "$CI_PROJECT_DIR/stackpacks/$STACKPACK" || {
        echo "ERROR: Stackpack validation failed for stackpacks/$STACKPACK"
        exit 1
      }
      echo "Stackpack validation passed for stackpacks/$STACKPACK"
  dependencies: []

validate-my-stackpack:
  <<: *validate-stackpack
  variables:
    STACKPACK: my-stackpack
----

===== Dockerコマンドライン

ローカルまたは他のCI/CDシステムで検証を実行する場合は、`+docker run+` ：

- パッケージ化された`+.sts+` ファイルを検証する：
+
[source,bash]
----
docker run --rm --entrypoint="/opt/docker/bin/stack-pack-validator" \
    -v /path/to/my-stackpack-0.0.1.sts:/stackpack.sts \
    registry.rancher.com/stackstate/stackstate-server:<latest-tag> \
    -file /stackpack.sts
----

- stackpackディレクトリを検証する：
+
[source,bash]
----
docker run --rm --entrypoint="/opt/docker/bin/stack-pack-validator" \
    -v /path/to/my-stackpack:/stackpack \
    registry.rancher.com/stackstate/stackstate-server:<latest-tag> \
    -directory /stackpack
----

[TIP]
====
`+stackstate-server+` イメージの`+stack-pack-validator+` スクリプトを CI/CD パイプラインで使用して、StackPacks を公開する前に検証します。これにより、稼働中の{stackstate-product-name} インスタンスを必要とせずに、StackPack がすべての要件を満たすようになります。
====

=== StackPackをアップロードする

`+upload+` サブコマンドは、実行中の{stackstate-product-name} インスタンスにzipアーカイブをプッシュする。

- アーカイブを{stackstate-product-name} にアップロードする：
+
[source,bash]
----
sts stackpack upload -f ./my-stackpack-0.0.1.zip
✅ ✓ Stackpack uploaded successfully!
----

=== StackPackをインストールする

{stackstate-product-name} stackpacks UIからxref:/stackpacks/about-stackpacks.adoc#_install_or_uninstall_a_stackpack[StackPackをインストールまたはアップグレード]します。
