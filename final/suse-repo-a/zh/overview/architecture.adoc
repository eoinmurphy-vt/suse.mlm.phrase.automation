=== 牧场经理

这是 Rancher 的核心组件，用户可以利用仪表板中现有的资源管理器功能访问群集工作负载详情。

=== 牧场主集群代理

代理部署在子群集中，使 Rancher 能够导入并与这些群集建立连接。通过这种连接，Rancher 可以在其平台内有效管理子集群。

=== {product_name} - 牧场主 CAPI 扩展

它提供 CAPI 和 Rancher 之间的集成，目前支持以下功能：

* *将 CAPI 群集导入 Rancher：*在 CAPI 配置的群集中安装 Rancher 群集代理。
* *CAPI 操作员配置：*为 CAPI 操作员提供配置支持。

== 部署拓扑

=== 牧场主经理与 CAPI 管理相结合

这是目前唯一支持的拓扑结构，其中 Rancher Manager 和{product_name} 都部署到同一个 Kubernetes 集群，并充当集中管理集群。

这种架构可简化组件的部署，并提供所有集群的单一视图。另一方面，必须考虑的是，群集 API (CAPI) 可有效管理的群集数量受到单个管理群集可用资源的限制。
