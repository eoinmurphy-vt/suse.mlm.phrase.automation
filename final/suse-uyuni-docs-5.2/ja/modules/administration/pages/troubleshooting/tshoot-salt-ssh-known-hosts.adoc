[[troubleshooting-salt-ssh-known-hosts]]
= Salt SSHの既知のホストのトラブルシューティング
:revdate: 2026-04-15
:page-revdate: {revdate}

マシンやVMが新しいSSHキーで再インストールされ、なおかつ同じIPアドレスで登録された場合、Saltが「中間者攻撃」について警告を発する可能性があります。
これは、Saltの`.ssh/known_hosts`ファイルに古いSSHフィンガープリントが残っているために発生します。

このリストからホストを削除するには、次のコマンドを実行します。

[source,shell]
----
mgrctl ssh knownhost remove <hostname>
----

削除するホストがデフォルトのSSHポート`22`を使用していない場合は、ポートを手動で指定できます。

[source,shell]
----
mgrctl ssh knownhost remove <hostname> <port>
----
