== {sl-micro} 6.1から{sl-micro} 6.2へ



この文書は、{sl-micro} 5.1サーバーでデプロイされた{productname} 6.1ホストを{sl-micro} 6.2にアップグレードし、{productname} {productnumber}に移行するためのテスト済み手順を提供します。


=== 前提条件

* {productname} 5.1は{sl-micro} 6.1にインストールされ、実行中です。
* システムは登録済みであり、{scc}にアクティブなサブスクリプションがあります。


=== ディストリビューションのアップグレードとサーバ移行

.プロシージャ:{productname} 5.1から{productname} {productnumber}への移行
[role=procedure]
____

. 現在の製品ステータスを確認する

+

[source,console]
----
SUSEConnect --status-text
----

+

確認:

* ベースOS: `SL Micro 6.1`
* 拡張機能: `SUSE Multi-Linux Manager Server 5.1 Extension`

+

. システムが更新されていることを確認する

+

[source,console]
----
transactional-update patch
----

+

--

* パッチが適用された場合、移行を続行する前に\\**システムを再起動**してください。

+

[source,console]
----
mgradm stop
reboot
----

* 更新が見つからなかった場合、移行ステップに直接進むことができます。

--

+

. 移行を開始する:

+

[source,console]
----
transactional-update migration --auto-agree-with-licenses --gpg-auto-import-keys
----

+

プロンプトに従って、**SUSE Linux Micro 6.2**および**{productname} サーバー拡張機能 {productnumber}**への利用可能な移行を選択します。

+

. 未使用のコンテナイメージをクリーンアップし、ディスク容量を解放します。

+
[source,shell]
----
podman image prune -a
----

+

ifndef::backend-pdf[]
include::installation-and-upgrade:partial$snippet-check-rpmnew-rpmsave.adoc[]
endif::[]

ifdef::backend-pdf[]
include::../../../../../partials/snippet-check-rpmnew-rpmsave.adoc[]
endif::[]

+

. サーバを停止し、再起動して変更を適用します。

+

[source,console]
----
mgradm stop
reboot
----

+

. 再起動後のチェック:

+

{productname}拡張機能と{productname}バージョンを確認します:

+

[source,console]
----
SUSEConnect --status-text
mgradm --version
----

+

予想される出力:

* 拡張機能: `SUSE Multi-Linux Manager Server {productnumber} Extension`
* バージョン: `mgradm version 5.2.0`以上

+

. サーバコンテナをアップグレードする

+

ifndef::backend-pdf[]
include::installation-and-upgrade:partial$snippet-ptf-loss-during-upgrade.adoc[]
endif::[]

ifdef::backend-pdf[]
include::../../../../../partials/snippet-ptf-loss-during-upgrade.adoc[]
endif::[]

+

[source,console]
----
mgradm start
mgradm upgrade podman
----

+

プロンプトに従い、新しい5.2.xコンテナをプルして設定します。


. PostgreSQLのバージョン

+

[source,console]
----
podman ps
----

+

予想される出力:

* `server:5.2.0`以上
* `server-postgresql:5.2.0`以上

+

. 未使用のコンテナイメージをクリーンアップし、ディスク容量を解放します。

+
[source,shell]
----
podman image prune -a
----

+

ifndef::backend-pdf[]
include::installation-and-upgrade:partial$snippet-check-rpmnew-rpmsave.adoc[]
endif::[]

ifdef::backend-pdf[]
include::../../../../../partials/snippet-check-rpmnew-rpmsave.adoc[]
endif::[]

+

. 実行中のコンテナを確認します。

+

[source,console]
----
podman ps
----

+

すべての予期されるプロキシコンテナが稼働中である必要があります。

____


=== 移行の完了

サーバーホストシステムは、{sl-micro}更新された{productname} {productnumber}サーバーパッケージを含む 6.2を実行しています。

このサーバーに接続されている{productname} 5.1プロキシがある場合は、xref:installation-and-upgrade:container-deployment/mlm/migrations/proxy/proxy-mlm-51-52.adoc[Proxy Migration 5.1 > 5.2]ガイドに進んでプロキシホストをアップグレードしてください。

生産運用を再開する前に、セットアップを確認してください。

== {sles} 15 SP7

この文書は、{productname} 5.1サーバーでデプロイされた{sles} 15 SP7ホストを{productname} {productnumber}にアップグレードするための手順を提供します。


=== 前提条件

* {productname} 5.1は{sles} 15 SP7にインストールされ、実行中です。
* システムは登録済みであり、{scc}にアクティブなサブスクリプションがあります。


=== アップグレードと移行

.プロシージャ:{productname} 15 SP7上の{sles}コンポーネントを更新します
[role=procedure]
____

. 現在の製品ステータスを確認する

+

[source,console]
----
SUSEConnect --status-text
----

+

確認:

* ベースOS: `SUSE Linux Enterprise Server 15 SP7`
* 拡張機能: `SUSE Multi-Linux Manager Server 5.1 Extension`

+

. システムが更新されていることを確認する

+

[source,console]
----
zypper patch
----

+

パッチが適用された場合、移行を続行する前に、サーバを停止し、システムを再起動してください。

+

[source,console]
----
mgradm stop
reboot
----

+

. {productname} {productnumber}への移行を実行します。

+

[source,console]
----
zypper migration
----

+

タイムゾーンを選択します。

* `{productname} Server Extension {productnumber}`

+

. サーバを停止し、再起動して変更を適用します。

+

[source,console]
----
mgradm stop
reboot
----

+

. 再起動後のチェック:

+

{productname} 拡張機能を確認してください：

+

[source,console]
----
SUSEConnect --status-text
----

+

予想される出力:

* 拡張機能: `SUSE Multi-Linux Manager Server {productnumber} Extension`

+

. {productname} バージョンを確認してください。

+

[source,console]
----
mgradm --version
----

+

予想される出力:

* バージョン: `mgradm version 5.2.0`以上

+

. サーバコンテナをアップグレードする

+

ifndef::backend-pdf[]
include::installation-and-upgrade:partial$snippet-ptf-loss-during-upgrade.adoc[]
endif::[]

ifdef::backend-pdf[]
include::../../../../../partials/snippet-ptf-loss-during-upgrade.adoc[]
endif::[]

+

[source,console]
----
mgradm start
mgradm upgrade podman
----

+

プロンプトに従い、新しい5.2.xコンテナをプルして設定します。


. コンテナの更新

+

[source,console]
----
podman ps
----

+

予想される出力:

* `server:5.2.0`以上
* `server-postgresql:5.2.0`以上

____


=== 移行の完了

サーバーホストシステムは現在、{sles} 15 SP7で動作しており、更新された{productname}{productnumber}サーバーパッケージが適用されています。

このサーバーに接続されている{productname} 5.1プロキシがある場合は、xref:installation-and-upgrade:container-deployment/mlm/migrations/proxy/proxy-mlm-51-52.adoc[Proxy Migration 5.1 > 5.2]ガイドに進んでプロキシホストをアップグレードしてください。

生産運用を再開する前に、セットアップを確認してください。
