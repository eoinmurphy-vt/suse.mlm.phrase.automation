== {sle-micro} 5.5から{sl-micro} 6.2



本書は、{sle-micro} 5.5ホスト（{productname} 5.0サーバーでデプロイされたもの）を{sl-micro} 6.2にアップグレードし、{productname} {productnumber}へ移行するためのテスト済み手順を提供します。


=== 前提条件

* {productname} 5.0は{sle-micro} 5.5にインストールされ、実行中です。
* システムは登録済みであり、{scc}にアクティブなサブスクリプションがあります。


=== ディストリビューションのアップグレードとサーバ移行

.プロシージャ:{productname} 5.0から{productname} {productnumber}への移行
[role=procedure]
____

. 現在の製品ステータスを確認する

+

[source,console]
----
SUSEConnect --status-text
----

+

確認:

* ベースOS: `SUSE Linux Enterprise Micro 5.5`
* 拡張機能: `SUSE Manager Server 5.0 Extension`

+

. システムが更新されていることを確認する

+

[source,console]
----
transactional-update patch
----

+

--

* パッチが適用された場合、移行を続行する前に\\**システムを再起動**してください。

+

[source,console]
----
mgradm stop
reboot
----

* 更新が見つからなかった場合、移行ステップに直接進むことができます。

--

+

. 移行を開始する:

+

[source,console]
----
transactional-update migration --auto-agree-with-licenses --gpg-auto-import-keys
----

+

プロンプトに従い、**SUSE Linux Micro 6.2** および **{productname} Server Extension {productnumber}** への移行先を選択してください。

+

. 未使用のコンテナイメージをクリーンアップし、ディスク容量を解放します。

+
[source,shell]
----
podman image prune -a
----

+

endif::[]

+

. サーバを停止し、再起動して変更を適用します。

+

[source,console]
----
mgradm stop
reboot
----

+

. 再起動後のチェック:

+

{productname}拡張機能と{productname}バージョンを確認してください:

+

[source,console]
----
SUSEConnect --status-text
mgradm --version
----

+

予想される出力:

* 拡張機能: `SUSE Multi-Linux Manager Server {productnumber} Extension`
* バージョン: `5.2.0`以上を参照

+

. PostgreSQLのバージョン

+

[source,console]
----
podman ps
----

+

予想される出力:

* `server:5.2.0`以上
* `server-postgresql:5.2.0`以上

+

. サーバコンテナをアップグレードする

+

include::installation-and-upgrade:partial$snippet-ptf-loss-during-upgrade.adoc[]

+

[WARNING]
====
`mgradm upgrade podman`を実行する前に、`/etc/systemd/system/uyuni-server.service`に`-p [::]:PORT:PORT`の形式で明示的なIPv6ポートバインディングが含まれているかどうかを確認してください。
これらのバインディングは{sl-micro} 6.2のnetavarkと互換性がなく、アップグレード後にWeb UIにアクセスできなくなる不正なnftablesルールを引き起こします。

そのような行が存在する場合は、続行する前に削除してください。

[source,diff]
----
  -p 80:80 \
- -p [::]:80:80 \
  -p 443:443 \
- -p [::]:443:443 \
----

次に、続行する前にホストを再起動してください:

[source,bash]
----
systemctl reboot
----
====

+

[source,console]
----
mgradm start
mgradm upgrade podman
----

+

プロンプトに従い、新しい5.2.xコンテナをプルして設定します。

+

. 実行中のコンテナを確認します。

+

[source,console]
----
podman ps
----

+

すべての予期されるプロキシコンテナが稼働中である必要があります。

. 未使用のコンテナイメージをクリーンアップし、ディスク容量を解放します。

+
[source,shell]
----
podman image prune -a
----

+

=== 移行の完了

サーバーホストシステムは、{sl-micro}更新された{productname} {productnumber}パッケージを使用して 6.2を実行しています。

このサーバーに接続されている{productname} 5.0プロキシがある場合は、xref:installation-and-upgrade:container-deployment/mlm/migrations/proxy/proxy-mlm-50-52.adoc[Proxy Migration 5.0 > 5.2]ガイドに進んでプロキシホストをアップグレードしてください。

生産運用を再開する前に、セットアップを確認してください。

== {sles} 15 SP6から15 SP7へ。

この文書は、{sles} 15 SP6ホスト（{productname} 5.0サーバーでデプロイされたもの）を{sles} 15 SP7にアップグレードし、{productname} {productnumber}へ移行するための手順を提供します。


=== 前提条件

* {productname} 5.0は{sles} 15 SP6にインストールされ、実行中です。
* システムは登録済みであり、{scc}にアクティブなサブスクリプションがあります。


=== ディストリビューションのアップグレードとサーバ移行

.プロシージャ:{productname} 15 SP6上の{sles}をアップグレードし、移行する
[role=procedure]
____

. 現在の製品ステータスを確認する

+

[source,console]
----
SUSEConnect --status-text
----

+

確認:

* ベースOS: `SUSE Linux Enterprise Server 15 SP6`
* 拡張機能: `SUSE Manager Server 5.0 Extension`

+

. システムが更新されていることを確認する

+

[source,console]
----
zypper patch
----

+

パッチが適用された場合、移行を続行する前に、サーバを停止し、システムを再起動してください。

+

[source,console]
----
mgradm stop
reboot
----

+

. SP7への移行と{productname} {productnumber}を実行します。

+

[source,console]
----
zypper migration
----

+

タイムゾーンを選択します。

* `{sles} 15 SP7`
* `{productname} Server Extension {productnumber}`

+

. サーバを停止し、再起動して変更を適用します。

+

[source,console]
----
mgradm stop
reboot
----

+

. 再起動後のチェック:

+

{productname} 拡張機能を確認してください：

+

[source,console]
----
SUSEConnect --status-text
----

+

予想される出力:

* 拡張機能: `SUSE Multi-Linux Manager Server {productnumber} Extension`

+

. {productname} バージョンを確認してください。

+

[source,console]
----
mgradm --version
----

+

予想される出力:

* バージョン: `5.2.0`以上を参照

+

. サーバコンテナをアップグレードする

+

ifndef::backend-pdf[]
include::installation-and-upgrade:partial$snippet-ptf-loss-during-upgrade.adoc[]
endif::[]

ifdef::backend-pdf[]
include::../../../../../partials/snippet-ptf-loss-during-upgrade.adoc[]
endif::[]

+

[source,console]
----
mgradm start
mgradm upgrade podman
----

+

プロンプトに従い、新しい5.2.xコンテナをプルして設定します。


. コンテナの更新

+

[source,console]
----
podman ps
----

+

予想される出力:

* `server:5.2.0`以上
* `server-postgresql:5.2.0`以上

____


=== 移行の完了

サーバーホストシステムは現在、{sles} 15 SP7 で動作しており、更新された{productname} {productnumber}パッケージが適用されています。

このサーバーに接続されている {productname} 5.0 プロキシがある場合は、xref:installation-and-upgrade:container-deployment/mlm/migrations/proxy/proxy-mlm-50-52.adoc[Proxy Migration 5.0 > 5.2] ガイドに進んでプロキシホストをアップグレードしてください。

生産運用を再開する前に、セットアップを確認してください。
