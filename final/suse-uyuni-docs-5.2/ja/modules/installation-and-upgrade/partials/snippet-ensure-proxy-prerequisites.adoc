:description: プロキシとして使用するホストの準備方法を学ぶ
== [package]\``podman``をインストールおよび有効にする

プロキシを使用する前に、いくつかのパッケージがホスト上に存在し、`podman`が実行されている必要があります。


.プロシージャ:前提条件の準備
[role=procedure]
_____

. プロキシホストで、次のコマンドを使用してプロキシをインストールします。

* `podman`
* `mgrpxy-bash-completion`
* `suse-multi-linux-manager-<version>-<arch>-proxy-httpd-image`
* `suse-multi-linux-manager-<version>-<arch>-proxy-salt-broker-image`
* `suse-multi-linux-manager-<version>-<arch>-proxy-squid-image`
* `suse-multi-linux-manager-<version>-<arch>-proxy-ssh-image`
* `suse-multi-linux-manager-<version>-<arch>-proxy-tftpd-image`

+

例えば、バージョン5.2、アーキテクチャx86_64の場合、パッケージ名は`suse-multi-linux-manager-5.1-x86_64-proxy-httpd-image`となります。

+

. システムを再起動するか、次のコマンドを実行してPodmanサービスを起動します。

+

[source, shell]
----
systemctl enable --now podman.service
----

. プロキシホストで、次のコマンドを使用してプロキシをインストールします。

+

----
mgrpxy install podman config.tar.gz
----

_____
