마이그레이션 과정 중에 서버 SSL 인증서와 CA 체인은 소스 서버에서 복사되므로, 데이터베이스 인증서만 필요합니다.



== {sl-micro} 6.1에서 {sl-micro} 6.2로



이 문서는 {sl-micro} 5.1 서버에서 배포된 {productname} 6.1 호스트를 {sl-micro} 6.2로 업그레이드하고 {productname} {productnumber}(으)로 마이그레이션하는 검증된 절차를 제공합니다.


=== 전제 조건

* {productname} 5.1이 {sl-micro} 6.1에 설치되어 실행 중입니다.
* 시스템은 {scc}에 등록되어 있으며 활성 구독을 보유하고 있습니다.


=== 업그레이드 및 마이그레이션

.절차 {productname} 5.1에서 {productname} {productnumber}(으)로의 마이그레이션
[role=procedure]
____

. 현재 제품 상태를 확인합니다.

+

[source,console]
----
SUSEConnect --status-text
----

+

확인:

* Base OS: `SL Micro 6.1`
* 확장 프로그램: `SUSE Multi-Linux Manager Server 5.1 Extension`

+

. 시스템이 업데이트되었는지 확인합니다.

+

[source,console]
----
transactional-update patch
----

+

--

* 패치가 적용된 경우 마이그레이션을 진행하기 전에 서버를 중지한 다음 시스템을 재부팅합니다.

+

[source,console]
----
mgradm stop
reboot
----

* 업데이트가 없는 경우, 마이그레이션 단계로 바로 진행할 수 있습니다.

--

+

. 업그레이드를 시작합니다.

+

[source,console]
----
transactional-update migration --auto-agree-with-licenses --gpg-auto-import-keys
----

+

안내에 따라 **SUSE Linux Micro 6.2** 및 **{productname} 서버 확장 프로그램 {productnumber}**(으)로 가능한 마이그레이션을 선택합니다.

+

. 사용하지 않는 컨테이너 이미지를 정리하여 디스크 공간을 확보합니다.

+
[source,shell]
----
podman image prune -a
----

+

+

. 서버를 중지한 다음 재부팅하여 변경 사항을 적용합니다.

+

[source,console]
----
mgradm stop

----

+

. 재부팅 후 점검을 수행합니다.

+

{productname} 확장 프로그램 및 {productname} 버전 확인:

+

[source,console]
----
SUSEConnect --status-text
mgradm --version
----

+

예상 출력:

* 확장 프로그램: `SUSE Multi-Linux Manager Server {productnumber} Extension`
* 버전: `mgradm version 5.2.0` 이상

+

[source,console]

=== 마이그레이션

서버 호스트 시스템이 이제 업데이트된 {sl-micro} {productname} 서버 패키지와 함께 {productnumber} 6.2를 실행 중입니다.

이 서버에 연결된 {productname} 5.1 프록시가 있는 경우, xref:installation-and-upgrade:container-deployment/mlm/migrations/proxy/proxy-mlm-51-52.adoc[Proxy Migration 5.1 > 5.2] 가이드로 진행하여 프록시 호스트를 업그레이드하십시오.

생산 작업을 재개하기 전에 설정을 검증하십시오.

== {sles} 15 SP7

이 문서는 {sles} 5.1 서버에서 배포된 {productname} 15 SP7 호스트를 {productname} {productnumber}(으)로 업그레이드하는 절차를 제공합니다.


=== 전제 조건

* {productname} 5.1이 {sles} 15 SP7에 설치되어 실행 중입니다.
* 시스템은 {scc}에 등록되어 있으며 활성 구독을 보유하고 있습니다.


=== 업그레이드 및 마이그레이션

.절차 {productname} 15 SP7에서 {sles} 구성 요소를 업데이트하십시오.
[role=procedure]
____

. 현재 제품 상태를 확인합니다.

+

[source,console]
----
SUSEConnect --status-text
----

+

확인:

* Base OS: `SUSE Linux Enterprise Server 15 SP7`
* 확장 프로그램: `SUSE Multi-Linux Manager Server 5.1 Extension`

+

. 시스템이 업데이트되었는지 확인합니다.

+

[source,console]
----

----

+

[source,console]

----

+

[source,console]

----

+

* `{productname} Server Extension {productnumber}`

+

. 서버를 중지한 다음 재부팅하여 변경 사항을 적용합니다.

+

[source,console]
----
mgradm stop

----

+

[source,console]

----

+

예상 출력:

* 확장 프로그램: `SUSE Multi-Linux Manager Server {productnumber} Extension`

+

[source,console]

mgradm --version

예상 출력:
