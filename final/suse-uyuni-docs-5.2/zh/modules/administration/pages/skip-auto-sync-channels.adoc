[[configure-autosync-channel]]
= 配置自动频道同步
:revdate: 2026-05-08
:page-revdate: {revdate}

默认情况下，频道会自动与上游存储库同步，以确保客户端能够访问最新的内容和更新。
但在某些情况下，您可能希望针对特定频道禁用此自动同步。

本章介绍了如何禁用频道的自动同步。




[[get-channel-sync-satus]]
== 验证频道同步状态

用户可以调用 API 来检查频道是否设置为自动同步。

要使用 API 检查频道的同步状态，请使用以下命令。
将 `<Channel_Label>` 替换为您要检查的频道的实际标签

[source,console]
----
spacecmd api -- --args "<Channel_Label>" channel.software.isAutoSync
INFO: Connected to http://localhost/rpc/api as admin
[
  true
]
----

如果启用了自动同步，该命令将返回 `true`；如果禁用，则返回 `false`。

此信息也可在 {webui} 中找到。
要检查同步状态，请导航至频道详细信息页面并查找 `Automatic Repository Synchronization` 字段。


[[set-channel-sync-satus]]
== 设置频道跳过自动同步

要启用或禁用频道的自动同步，您可以使用 API 调用 `channel.software.setAutoSync`。

以下是如何使用 API 设置频道跳过自动同步的示例（将 `<Channel_Label>` 替换为您要更新的频道的实际标签）：

[source,console]
----
spacecmd api -- --args '["<Channel_Label>", false]' channel.software.setAutoSync
INFO: Connected to http://localhost/rpc/api as admin
[
  false
]
----

要重新启用自动同步，请将第二个参数设置为 `true`：

[source,console]
----
spacecmd api -- --args '["<Channel_Label>", true]' channel.software.setAutoSync
INFO: Connected to http://localhost/rpc/api as admin
[
  true
]
----


[[manual-channel-sync]]
== 手动同步频道

即使您已禁用频道的自动同步，您仍然可以在需要时手动同步它。

[IMPORTANT]
====
在服务器容器内运行步骤之前，请使用 `mgrctl term`。
====

要手动同步频道，您可以使用该命令（将 `<Channel_Label>` 替换为您要同步的频道的实际标签）：

[source,console]
----
spacewalk-repo-sync -c <Channel_Label>
----
